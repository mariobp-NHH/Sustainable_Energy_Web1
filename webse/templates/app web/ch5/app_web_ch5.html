Chapter 5. Deploy

In this chapter, we deploy our app. First, we prepare a folder in our computer where we place the information required to deploy our app. Second, we create a repository in GitHub where we will move the folder in our computer where we have developed our app. Third, we connect our GitHub repository with our Heroku account to deploy our app in the server.

Folder in the computer

We follow the next steps to have all the files required to deploy our app.

  1. From OneDrive, download the entire folder (Ch5. Deploy).
    OneDrive, Ch5. Deploy
  2. That folder contains the next information:
    • The recipe of the cake: static folder, templates folder and app.py file
    • The instructions to create the tools in the server to bake the cake: Procfile and requirements files.

GitHub

We create a repository in GitHub where we locate all the files required to deploy our app. We proceed in three steps.

  1. We create an account in GitHub.
  2. We create a new repository.
    1. Name of the repository: ENE425-2022-app-yourname
    2. Add a readme file
    3. Choose a license. I propose you to use the MIT license, but you can use the one that you prefer.
  3. Add files to the repository
    1. Drag all the files from the folder in your computer.
    2. Commit the changes

Heroku

Deploy the app in Heroku by following three steps.

  1. Create a new app, and name it: ENE425-2022-app-yourname
  2. Deploy the app:
    1. Connect to GitHub by using the repo-name that you used in GitHub: ENE425-2022-app-yourname
    2. Chick the button search, and once that Heroku finds your repository, click the button connect.
    3. Click the button enable automatic deploys to upload instantaneously the files that you upload in your GitHub repository.
    4. Click the button deploy branch.
  3. Click the button open the app to open your app in the browser.
